Sessions in Baboon

Sessions werden in Baboon ausschließlich vom Server mit folgenden Optionen erzeugt und verwaltet.

config/app.conf.json

"sessionKey": "baboon.sid" ,
"sessionSecret": "a7f4eb39-744e-43e3-a30b-3ffea846030f",
"sessionMaxLife": 804600,
"sessionInactiveTime": 3600

In der Option sessionKey beschreibt den Namen des Cookies.
In der Option sessionSecret wird der Wert zum Verschlüsseln der Session angegeben.
Die Option sessionMaxLife beschreibt die Maximale Laufzeit einer Session. Dabei ist diese Zeit absolut, nach der angegebenen Zeit wird die Session definitiv gelöscht unabhängig von der Aktivität.
Die Option sessionInactiveTime gibt die Zeit an in welcher der Benutzer inaktiv sein kann. Die Session wird beim Überschreiten dieser Zeitspanne zerstört.

Da dies vom Server gemacht wird, muss es eine Möglichkeit für den Client geben seine Aktivität an den Server zu melden. Dies macht Baboon über das $routeChangeStart Event von Angular in der app.js. 

app.js

$rootScope.$on('$routeChangeStart', function () {
            session.setActivity();
        });

Das bedeutet, bei jeder Änderung der Route meldet der Client eine Aktivität an den Server. Bei dieser Meldung wird die Session auch auf die Optionen: sessionMaxLife und sessionInactiveTime geprüft. Ist die Session abgelaufen wird die Session zerstört und der Benutzer Automatisch abgemeldet.
